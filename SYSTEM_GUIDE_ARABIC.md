# 📋 دليل نظام مواجهة الشائعات - النسخة المتقدمة

## 🎯 نظرة عامة على النظام

نظام متكامل لمواجهة الشائعات وحروب الجيل الباردة بدعم كامل للغة العربية وتركيز على مصر والدول العربية.

## 🏛️ الأدوار والصلاحيات

### 1️⃣ **المستخدم العادي** (User)
- ✅ رفع بلاغات جديدة مع أدلة ملفية
- ✅ عرض بلاغاته الشخصية وحالتها
- ✅ البحث في قاعدة البيانات
- ✅ قراءة المقالات التوعوية
- ✅ الوصول للروابط الموثوقة

### 2️⃣ **المراجع** (Reviewer)
- ✅ عرض جميع البلاغات المعلقة
- ✅ مراجعة وتقييم البلاغات
- ✅ إضافة تعليقات ومرفقات
- ✅ الموافقة/الرفض على البلاغات
- ✅ جميع صلاحيات المستخدم العادي

### 3️⃣ **المدير** (Admin)
- ✅ جميع صلاحيات المراجع
- ✅ إدارة لوحة التحكم
- ✅ تحديث الروابط الخارجية الموثوقة
- ✅ إدارة الفئات والإعدادات
- ✅ عرض التحليلات المتقدمة
- ✅ إدارة المستخدمين والأدوار

### 4️⃣ **المتحدث الإعلامي** (Spokesperson)
- ✅ عرض البلاغات المحققة
- ✅ إصدار البيانات الإعلامية الرسمية
- ✅ تحديد النتيجة النهائية (صحيح/خاطئ)
- ✅ نشر الحكم النهائي بشكل رسمي

## 📊 نظام البلاغات المتقدم (Advanced Report System)

### مراحل البلاغ

```
المستخدم يرفع بلاغ
    ↓
المراجع يقيم البلاغ (يضيف تعليقه)
    ↓
المدير يراجع رأي المراجع (يعلّق/يوافق)
    ↓
المتحدث الإعلامي يصدر البيان الرسمي
    ↓
النشر النهائي (صحيح/خاطئ/قيد المراجعة)
```

### حالات البلاغات

| الحالة | الوصف |
|--------|-------|
| **معلق (Pending)** | في انتظار المراجع |
| **موافق (Approved)** | تمت الموافقة من المرحلة |
| **مرفوض (Rejected)** | تم الرفض ولا يتقدم |
| **صحيح (True)** | تم التحقق - الخبر صحيح |
| **خاطئ (False)** | تم التحقق - الخبر خاطئ |
| **قيد المراجعة (Unverified)** | لا يزال تحت التحقق |

## 💻 كيفية استخدام النظام

### للمستخدم العادي

#### 1. إنشاء حساب جديد
```
زر "تسجيل" → ملء البيانات (الاسم، البريد، كلمة المرور) → تأكيد
```

#### 2. رفع بلاغ جديد
```
الرئيسية → "رفع بلاغ" → 
  - أدخل عنوان البلاغ
  - صِف الشائعة بالتفصيل
  - أضف أدلة ملفية (صور، فيديو، PDF)
  - حدد التصنيف
  - أضف وسوم (تاجات)
  → "إرسال البلاغ"
```

#### 3. تتبع البلاغات
```
"بلاغاتي" → عرض حالة كل بلاغ →
  - معلق: في انتظار المراجع
  - تحت المراجعة: المراجع يعمل عليه
  - موافق: سيتقدم للمدير
  - مرفوض: لم يتم قبوله
  - صحيح/خاطئ: تم الحكم النهائي
```

#### 4. البحث والاطلاع
```
"البحث" → اكتب الكلمة → عرض النتائج المشابهة
"الروابط الموثوقة" → اختر الجهة → اذهب للموقع
```

---

### للمراجع (Reviewer)

#### 1. الوصول لقائمة البلاغات
```
التنقل العلوي → "فريق التحقق" → عرض البلاغات المعلقة
```

#### 2. مراجعة بلاغ
```
"فريق التحقق" → اختر بلاغ → "عرض التفاصيل" →
  - اقرأ التفاصيل والأدلة
  - انقر "تقييم المراجع" → اختر الحالة:
    ✓ موافق: هذا البلاغ موثوق
    ✗ مرفوض: لا يستحق المتابعة
  - أضف تعليقك
  → "إرسال تقييم المراجع"
```

---

### للمدير (Admin)

#### 1. الوصول لوحة التحكم
```
التنقل العلوي → "لوحة المدير"
```

#### 2. مزامنة الإعدادات والروابط
```
لوحة المدير →
  - عدل اسم الموقع والوصف الأساسي
  - أضف/عدل/حذف الروابط الخارجية الموثوقة
  - أضف فئات جديدة
  → "حفظ الإعدادات"
```

#### 3. إضافة رابط خارجي
```
"لوحة المدير" → تمرير لأسفل → "الروابط الخارجية الموثوقة" →
  - اسم الرابط: "وزارة الداخلية المصرية"
  - الرابط: "https://www.moig.gov.eg"
  - الوصف: "الموقع الرسمي لوزارة الداخلية"
  - الأيقونة: "🏛️"
  - التصنيف: "حكومي مصري"
  → "إضافة رابط موثوق"
```

#### 4. مراجعة تقييمات المراجعين
```
"فريق التحقق" → اختر بلاغ →
  - اطلع على تقييم المراجع
  - أضف تعليقك كمدير
  - اختر الحالة (موافق/مرفوض)
  → "إرسال تقييم المدير"
```

#### 5. عرض التحليلات
```
"التحليلات المتقدمة" → 
  - إحصائيات البلاغات
  - أكثر الفئات تقارير
  - الوسوم الأكثر استخداماً
```

---

### للمتحدث الإعلامي (Spokesperson)

#### 1. الوصول لنظام المراجعة
```
التنقل العلوي → "فريق التحقق"
```

#### 2. إصدار البيان الإعلامي النهائي
```
"فريق التحقق" → اختر بلاغ موثوق →
  - اطلع على تقييمات المراجع والمدير
  - اختر الحكم النهائي:
    ✓ صحيح: الخبر صحيح ومؤكد
    ✗ خاطئ: الخبر كاذب أو مضلل
    ⏳ قيد المراجعة: تحتاج معلومات إضافية
  - اكتب البيان الإعلامي الرسمي:
    (مثال: "هذا الخبر غير صحيح. الحقيقة أن...")
  → "نشر الحكم النهائي والبيان الإعلامي"
```

## 🔧 النقاط التقنية المهمة

### قاعدة البيانات الجديدة:
```sql
CREATE TABLE report_reviews (
  review_id INT PRIMARY KEY AUTO_INCREMENT,
  report_id INT NOT NULL,
  reviewer_id INT,
  admin_id INT,
  spokesperson_id INT,
  reviewer_comment TEXT,
  reviewer_status ENUM('pending', 'approved', 'rejected'),
  admin_comment TEXT,
  admin_status ENUM('pending', 'approved', 'rejected'),
  spokesperson_comment TEXT,
  spokesperson_status ENUM('pending', 'approved', 'rejected'),
  final_verdict ENUM('true', 'false', 'unverified'),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### API Endpoints الجديدة:
- `GET /api/reports-with-reviews` - عرض البلاغات مع المراجعات
- `POST /api/reports/:id/review` - إنشاء مراجعة
- `PUT /api/reports/:id/reviewer-assessment` - تقييم المراجع
- `PUT /api/reports/:id/admin-assessment` - تقييم المدير
- `PUT /api/reports/:id/spokesperson-assessment` - تقييم المتحدث
- `GET /api/reports/:id/full-review` - البيانات الكاملة
- `GET /api/external-links` - الروابط الخارجية
