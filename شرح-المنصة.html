1. نظرة عامة على المشروع وأهميته

منصة مكافحة الشائعات هي تطبيق ويب متكامل تم تطويره لمواجهة ظاهرة الأخبار الزائفة والمعلومات المضللة في المجتمع المصري والعربي، خاصةً في إطار ما يُعرف بـ "حروب الجيل الرابع" — وهي الحروب الإعلامية والمعلوماتية التي تستهدف إضعاف المجتمعات من الداخل عبر نشر الذعر، التفرقة، وتشويه الحقائق.

💡 أهمية المشروع:
تُشير الإحصاءات إلى أن 68% من مستخدمي الإنترنت في مصر يتعرضون لأخبار زائفة يومياً عبر وسائل التواصل الاجتماعي. هذه المنصة توفر أداة مجتمعية وتقنية لمواجهة هذه الظاهرة بالتحليل الذكي والتوعية.

الأهداف الرئيسية للمنصة:

  • 🔍 التحقق الآلي — تحليل الأخبار والشائعات باستخدام Gemini AI لتحديد مصداقيتها فورياً
  • 📢 الإبلاغ المجتمعي — تمكين المواطنين من الإبلاغ عن الشائعات التي يتعرضون لها
  • 📰 التوعية والتثقيف — نشر مقالات ومحتوى توعوي حول كيفية التعرف على الأخبار الزائفة
  • 🎬 الفيديوهات التوعوية — عرض مقاطع فيديو من YouTube وFacebook وTikTok لنشر الوعي
  • 🤖 المساعد الذكي — محادثة مباشرة مع Gemini AI للاستشارة حول أي خبر أو معلومة
  • ⚙️ لوحة إدارة — تحكم كامل للمشرفين في المحتوى والمستخدمين والبلاغات

شرح كل صفحة بالتفصيل:

📢 صفحة الإبلاغ — /report

  • نموذج يتضمن: العنوان، التفاصيل، رابط المصدر، التصنيف (صحة/سياسة/اقتصاد/أمن/دين/عام)، الوسوم
  • يرسل البيانات إلى POST /api/reports
  • يدعم رفع ملفات كأدلة (صور، PDF) — حتى 5 ملفات × 10MB
  • يظهر رسالة نجاح خضراء بعد الإرسال

📰 صفحة المقالات — /articles

  • تجلب المقالات من GET /api/articles في الوقت الفعلي
  • بحث فوري في العنوان والمحتوى
  • عرض بطاقات Grid تحتوي: الفئة، العنوان، مقتطف، المؤلف، التاريخ

🔍 صفحة التحقق AI — /ai-detect

  • حقل نصي لإدخال الخبر أو الشائعة
  • يرسل إلى POST /api/verify-ai
  • يعرض: نسبة المصداقية (0-100%)، حكم (صحيح/مشبوه/شائعة)، ملخص، أسباب
  • إذا توفر GEMINI_API_KEY: تحليل حقيقي بـ Google Gemini 1.5 Flash
  • إذا لم يتوفر: تحليل محلي بناءً على كلمات الشك

🤖 صفحة الدردشة AI — /ai-chat

  • واجهة دردشة تفاعلية مثل WhatsApp
  • يرسل رسائل إلى POST /api/ai-chat
  • Gemini يجيب بالعربية عن أي سؤال يخص الشائعات والمعلومات المضللة
  • يحفظ سجل المحادثة خلال الجلسة الواحدة

🎬 صفحة الفيديوهات — /videos

  • يجلب الفيديوهات من GET /api/videos
  • YouTube: يعرض Player مدمج iframe مباشرة
  • Facebook / TikTok / أخرى: زر للانتقال إلى الرابط
  • المشرف فقط: يرى زر "+ إضافة فيديو" + زر "حذف" على كل فيديو

⚙️ لوحة التحكم — /admin

  • 4 بطاقات إحصاء: إجمالي البلاغات، قيد المراجعة، المقالات، المصادر
  • جدول أحدث 5 بلاغات مع حالتها
  • يجلب من GET /api/dashboard و GET /api/reports

📝 إدارة المقالات — /admin-articles

  • CRUD كامل: إضافة / تعديل / حذف المقالات
  • حقول: العنوان، المحتوى، الفئة، الوسوم، حالة النشر
  • جدول بجميع المقالات مع إجراءات لكل صف

مزامنة البيانات (Data Synchronization):

✅ آلية المزامنة في المشروع:
1. كل طلب Frontend يستخدم Axios Instance مع baseURL ثابت
2. الـ JWT token يُرسل تلقائياً مع كل طلب عبر Request Interceptor
3. قاعدة البيانات على Railway تعمل 24/7 مع connection pooling
4. صفحة المقالات والفيديوهات تجلب البيانات في Real-Time عند كل زيارة
5. لوحة التحكم تجلب الإحصائيات من الـ API مباشرةً (لا cache محلي للإحصائيات)
6. contentSettings.js تحفظ في localStorage وتُزامَن مع واجهة المشرف

7. الذكاء الاصطناعي — Gemini Integration

نموذج الذكاء الاصطناعي:

المنصة تستخدم Google Gemini 1.5 Flash عبر الـ REST API الرسمي من Google.

التحقق من الشائعات (/api/verify-ai):

يُرسل الـ Backend prompt محكم إلى Gemini يطلب منه تحليل النص وإرجاع JSON بـ:

  • confidence: نسبة صحة المعلومة (0-100)
  • verdict: صحيح / مشبوه / شائعة
  • summary: ملخص التحليل بالعربية
  • reasons: قائمة أسباب الحكم

Fallback بدون API Key:

إذا لم يُضف GEMINI_API_KEY في Railway، يعمل النظام بـتحليل محلي يبحث عن كلمات مثل: "عاجل"، "خطير"، "انتشار"، "خبر مهم"، إلخ — لكن الدقة محدودة. يُنصح بإضافة المفتاح للحصول على تحليل حقيقي.

الحصول على GEMINI_API_KEY:

  1. افتح https://aistudio.google.com/app/apikey
  2. اضغط "Create API Key"
  3. انسخ المفتاح
  4. اذهب إلى Railway → backend service → Variables → أضف: GEMINI_API_KEY = مفتاحك

8. نظام المصادقة والصلاحيات

تدفق تسجيل الدخول:

المستخدم يدخل email + password
        ↓
Frontend يرسل POST /api/login
        ↓
Backend يتحقق من البريد في DB
        ↓
bcrypt.compare(password, hash)
        ↓
jwt.sign({ userId, email, role, name }, SECRET, { expiresIn: '7d' })
        ↓
يُرجع: { token, role, userId, name }
        ↓
Frontend يحفظ في localStorage:
  token | role | name | userId
        ↓
Axios interceptor يضيف تلقائياً:
  Authorization: Bearer <token>

مستويات الصلاحيات:

الدورالصلاحيات
userتسجيل دخول، إرسال بلاغ، قراءة مقالات، التحقق AI، دردشة AI، مشاهدة فيديوهات
reviewerكل صلاحيات user + مراجعة البلاغات وتحديث حالتها
adminكل الصلاحيات + إدارة المقالات، الفيديوهات، المستخدمين، لوحة التحكم الكاملة
spokespersonدور خاص للمتحدث الرسمي للمراجعة والتعليق

حماية الـ Routes:

  • Frontend: isAuthenticated يتحقق من وجود token في localStorage
  • Frontend: isAdmin يتحقق أن role === 'admin'
  • Backend: Middleware auth يتحقق من JWT في كل طلب محمي
  • Backend: Middleware admin يتحقق أن role === 'admin'

12. دليل المشرف

🔑 بيانات الدخول الافتراضية للمشرف:
البريد: admin@platform.com
كلمة المرور: admin123
يُنصح بتغيير كلمة المرور بعد أول دخول.

ما يستطيع المشرف فعله:

  1. إضافة مقالات توعوية — عبر /admin-articles → "+ مقال جديد"
  2. تعديل وحذف المقالات — من نفس الصفحة
  3. إضافة فيديوهات — عبر /videos → "+ إضافة فيديو" (يدعم YouTube/Facebook/TikTok)
  4. مراجعة البلاغات — عبر /admin → جدول أحدث البلاغات
  5. مشاهدة الإحصائيات — عبر /admin → 4 بطاقات إحصائية
  6. تغيير أدوار المستخدمين — عبر API: PUT /api/admin/users/:id/role

إضافة Gemini AI (خطوة واحدة):

  1. اذهب إلى: https://railway.com → تسجيل دخول
  2. اختر المشروع → خدمة الـ Backend → Variables
  3. أضف: GEMINI_API_KEY = مفتاحك من aistudio.google.com
  4. Railway يُعيد النشر تلقائياً خلال دقيقتين
  5. الآن تحقق AI ودردشة AI يعملان بالذكاء الاصطناعي الحقيقي
